## 一、结缘开源

## 三、通义灵码体验
虽然接触过 CI/CD，但是我从来没有在 Github 上实现过它，也不会 `Github Action `的语法格式。通义灵码给了我很大的帮助。我按照通义灵码安装文档将它安装到了我的 VS Code 中，接下来就是愉快的使用环节了。<br />在使用通义灵码的过程中，我先复制了一份写好的 `Github Action Workflow`，利用通义灵码的**解释代码**和**直接对话**功能，我很快地了解了 `Github Action `的基本语法以及格式。<br />![](https://intranetproxy.alipay.com/skylark/lark/0/2024/png/299576/1723427330771-8a65a554-75c3-402f-883f-10688c59549d.png#clientId=u27032a0a-91c4-4&from=paste&id=ua35067a4&originHeight=573&originWidth=1080&originalType=url&ratio=1.7999999523162842&rotation=0&showTitle=false&status=done&style=none&taskId=u50bdd414-2e72-495e-be26-62e02951489&title=)<br />![](https://intranetproxy.alipay.com/skylark/lark/0/2024/png/299576/1723427331300-0d64b720-49c8-49b3-9676-4c9f0dd38eeb.png#clientId=u27032a0a-91c4-4&from=paste&id=u934f3feb&originHeight=577&originWidth=1080&originalType=url&ratio=1.7999999523162842&rotation=0&showTitle=false&status=done&style=none&taskId=u9c3cb495-e927-4dcf-bdf3-d2aa234c4ad&title=)<br />然后我就开始编写 `issue `所需要的 `Github Action Workflow `了。<br />其中**代码优化建议**功能帮了我很大的忙，我只需要把有疑问的代码勾选，然后再执行 `/generate optimization`，通义灵码就能很方便的为我指出错误，并且提出建议。在以往的话我还需要先把代码复制下来，然后再抛给 AI，有时还得附带上下文。<br />![](https://intranetproxy.alipay.com/skylark/lark/0/2024/png/299576/1723427331278-07fb06bc-0133-4604-8bd0-e81f5d0bd698.png#clientId=u27032a0a-91c4-4&from=paste&id=uf70ea704&originHeight=576&originWidth=1080&originalType=url&ratio=1.7999999523162842&rotation=0&showTitle=false&status=done&style=none&taskId=u3fe5f274-d088-4559-a439-1e84b52e4c9&title=)

## 四、提交 PR

## 五、参赛心得
<a name="8bd66aa3"></a>
### 技术提升
通过这次比赛，我学会了`Github Action Workflow`的基本语法和编写格式，并且对 CI/CD 在企业中的应用有了更进一步的了解。<br />并且我还上手了阿里云开发的新一代 API 网关 Higress，体验了它丰富的功能，感受到了它在云原生环境下的强大实力。
<a name="e797f8d1"></a>
### 开源氛围
Higress 是一个活跃的开源社区，在这个社区中大家都积极的分享自己的知识和 idea，乐于寻找bug。这种互帮互助，乐于分享的活跃氛围大大激发了我的开源兴趣。
<a name="8e32cb89"></a>
### 个人体会
这次比赛不仅提升了我的技术，还让我学会了如何沟通、如何解决问题等等。我收获很多，相信在以后我会对开源社区做更多高质量的贡献，争取在多几个 PR~
